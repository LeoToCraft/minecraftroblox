# Minecraft: Roblox Edition

Roblox-проект песочницы в стиле Minecraft, собранный на Rojo. Игра включает меню миров, создание и загрузку сохранений, процедурную генерацию блочного мира, базовую систему разрушения/установки блоков, жидкости, падающие блоки, инвентарь, одиночный режим и заготовку мультиплеера через приватные серверы Roblox.

## Возможности

- Создание, переименование и удаление миров в 3 слотах.
- Размеры мира: Small, Medium, Large.
- Типы мира: Normal и Superflat.
- Генерация ландшафта по seed: холмы, горы, обрывы, реки, озера, лавовые озера, пещеры и деревья.
- Сохранение изменений мира: сломанные и поставленные блоки записываются в данные мира.
- Блоки: трава, земля, камень, булыжник, дерево, листья, песок, гравий, вода и лава.
- Физика песка и гравия: блоки падают вниз, если под ними нет опоры.
- Жидкости: вода и лава растекаются, а при контакте могут создавать булыжник.
- Клиентское меню, пауза, настройки, хотбар, инвентарь и прицел.
- Поддержка хостинга мира через Roblox Reserved Servers.

## Структура проекта

```text
src/
  client/
    ctrSettings.client.luau  # управление игроком, хотбар, инвентарь, ломание/установка блоков
    guiMenu.client.luau      # главное меню, список миров, настройки, мультиплеер, пауза
  server/
    wrlManager.server.luau   # слоты миров, сохранения, загрузка, хостинг, состояние игрока
    lqdSystem.server.luau    # генерация мира и базовое создание блоков
    blcManager.server.luau   # обновление жидкостей и взаимодействие воды/лавы
    plrModel.server.luau     # серверная проверка ломания и установки блоков
  shared/
    Hello.luau
```

## Архитектура

Основное состояние проекта хранится в глобальной таблице `_G.MCRE`. Серверные скрипты используют ее как общий runtime-контекст: `wrlManager` управляет жизненным циклом мира, `lqdSystem` строит мир, `plrModel` обрабатывает действия игрока, а `blcManager` периодически обновляет жидкости.

Связь клиента и сервера идет через объекты в `ReplicatedStorage.rmtGame`. Клиент вызывает RemoteFunction/RemoteEvent для списка миров, создания мира, загрузки, сохранения, настроек, хостинга, разрушения и установки блоков.

Шаблоны блоков ожидаются в `ServerStorage.blcStorage`. В Studio они должны называться:

- `GrassBlock`
- `DirtBlock`
- `StoneBlock`
- `CobblestoneBlock`
- `LogBlock`
- `LeavesBlock`
- `SandBlock`
- `GravelBlock`
- `WaterBlock`
- `LavaBlock`

При создании блоков в мире скрипты переименовывают их в игровые имена вида `blcGrass`, `blcDirt`, `blcWater` и т.д. Эти имена используются клиентом, логикой сохранений и обработкой взаимодействий.

## Запуск через Rojo

Собрать place-файл:

```bash
rojo build -o "minecraftroblox.rbxlx"
```

Запустить live-синхронизацию с Roblox Studio:

```bash
rojo serve
```

После этого открой проект в Roblox Studio и подключи Rojo-плагин к локальному серверу.

## Важные заметки

- Рабочая папка скриптов: `src`.
- Если в проекте есть папка `scripts`, она не является основной рабочей папкой этого проекта.
- Для корректной работы в Studio должны существовать `ReplicatedStorage.rmtGame`, `ReplicatedStorage.guiAssets` и `ServerStorage.blcStorage` с нужными ассетами.
- Если Output Roblox показывает `Infinite yield possible on 'ServerStorage:WaitForChild("...")'`, проверь совпадение имен папок и объектов в Studio с именами, указанными выше.
